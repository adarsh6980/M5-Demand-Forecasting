# 📦 M5 Demand Forecasting System

An end-to-end demand forecasting pipeline built on the [Walmart M5 Forecasting](https://www.kaggle.com/competitions/m5-forecasting-accuracy) dataset. Uses **XGBoost** with continuous learning, **concept drift detection**, and configurable **business rules** to produce actionable order recommendations.

---

## ✨ Features

| Capability | Description |
|---|---|
| **Data Cleaning** | Samples and transforms raw M5 wide-format data into daily POS time-series |
| **Feature Engineering** | Calendar, lag, and rolling-window features — no external weather data required |
| **XGBoost Forecasting** | Per-SKU models with time-series cross-validation and incremental retraining |
| **Drift Detection** | ADWIN + DDM detectors monitor prediction residuals for concept drift |
| **Business Rules** | YAML-driven constraints (shelf capacity, budget, perishability, safety stock) |
| **Console Pipeline** | Coloured terminal output with step-by-step progress and formatted results |

## 🚀 Getting Started

### Prerequisites

- Python 3.9+

### Installation

```bash
# Clone the repository
git clone https://github.com/adarsh6980/M5-Demand-Forecasting.git
cd M5-Demand-Forecasting

# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate   # macOS / Linux
# .venv\Scripts\activate    # Windows

# Install dependencies
pip install -r requirements.txt
```

### Running the Pipeline

```bash
python src/run_pipeline.py
```

The pipeline executes five steps in sequence:

1. **Data Cleaning** — loads raw M5 CSV files, samples top items from a target store, and writes `data/pos_data.csv`
2. **Feature Engineering** — adds calendar, lag, and rolling-window features
3. **Model Training** — trains per-SKU XGBoost models with time-series cross-validation
4. **Drift Detection** — evaluates prediction residuals with ADWIN and DDM detectors
5. **Business Rules** — applies capacity, budget, and perishability constraints to produce final order quantities

---

## ⚙️ Configuration

Business rules live in `config/business_rules.yml`. Each SKU entry supports:

| Parameter | Description |
|---|---|
| `max_shelf_capacity` | Maximum units the shelf can hold |
| `unit_cost` | Cost per unit (used for budget cap) |
| `max_budget_per_order` | Maximum spend per order |
| `perishability_days` | Shelf life in days |
| `safety_stock_days` | Minimum days of safety stock to maintain |

## 🧠 Model Details

- **Algorithm**: XGBoost Regressor
- **Validation**: Time-series cross-validation (5 splits)
- **Features**: day-of-week, month, week-of-year, weekend/holiday flags, promo flag, lags (1, 7, 14), rolling mean & std (7, 14)
- **Metrics**: MAE, RMSE, R²
- **Continuous Learning**: supports incremental retraining and tracks performance improvement over time

---

## 📊 Drift Detection

Two complementary detectors run on every prediction:

- **ADWIN** (Adaptive Windowing) — detects distributional changes in residual windows
- **DDM** (Drift Detection Method) — monitors error rate statistics for warning and drift levels

When drift is detected the pipeline reports severity, the triggering detector(s), and a human-readable explanation.
